The Crypto Market Analysis SaaS provides a comprehensive REST API built with Flask that exposes cryptocurrency predictions, market analysis, chat functionality, and administrative controls. This guide covers all available endpoints, authentication, request/response formats, and usage examples.

**Base URLs:**
- Local Development: `https://crypto-ai.local:10443/api`
- AWS Production: `https://crypto-ai.crypto-vision.com/api`

**API Version:** v1

## Authentication

All API endpoints require authentication using API keys passed in the request header.

### API Key Header

```http
X-API-Key: your-api-key-here
```

### Obtaining an API Key

API keys are generated by system administrators. Contact your admin to receive credentials.

### Authentication Example

```bash
curl -H "X-API-Key: your-api-key-here" \
     https://crypto-ai.local:10443/api/predictions/top20
```

### Authentication Errors

**401 Unauthorized:**
```json
{
  "error": "Unauthorized",
  "message": "Invalid or missing API key",
  "code": "AUTH_001"
}
```

## Rate Limiting

To ensure fair usage and system stability, the API implements rate limiting:

- **Rate:** 100 requests per minute per API key
- **Header Response:** Rate limit information is included in response headers

### Rate Limit Headers

```http
X-RateLimit-Limit: 100
X-RateLimit-Remaining: 95
X-RateLimit-Reset: 1698765432
```

### Rate Limit Exceeded

**429 Too Many Requests:**
```json
{
  "error": "Rate Limit Exceeded",
  "message": "You have exceeded the rate limit of 100 requests per minute",
  "code": "RATE_001",
  "retry_after": 45
}
```

## Error Handling

The API uses standard HTTP status codes and returns consistent error responses.

### HTTP Status Codes

| Code | Meaning | Description |
|------|---------|-------------|
| 200 | OK | Request succeeded |
| 201 | Created | Resource created successfully |
| 400 | Bad Request | Invalid request parameters |
| 401 | Unauthorized | Authentication failed |
| 403 | Forbidden | Insufficient permissions |
| 404 | Not Found | Resource not found |
| 429 | Too Many Requests | Rate limit exceeded |
| 500 | Internal Server Error | Server error occurred |
| 503 | Service Unavailable | Service temporarily unavailable |

### Error Response Format

```json
{
  "error": "Error Type",
  "message": "Human-readable error description",
  "code": "ERROR_CODE",
  "details": {
    "field": "Additional context"
  }
}
```

### Common Error Codes

| Code | Description |
|------|-------------|
| AUTH_001 | Invalid or missing API key |
| AUTH_002 | Insufficient permissions |
| RATE_001 | Rate limit exceeded |
| VAL_001 | Invalid request parameters |
| VAL_002 | Missing required field |
| PII_001 | PII detected in input |
| TOPIC_001 | Invalid topic (non-crypto) |
| DATA_001 | Insufficient data for prediction |
| EXT_001 | External API failure |

**Authentication:** Required

**Parameters:** None

**Response:**

```json
{
  "tendency": "bullish",
  "confidence": 0.78,
```

**Tendency Types:**
- `bullish`: Strong upward momentum (>60% cryptos increasing)
- `bearish`: Strong downward momentum (>60% cryptos decreasing)

- `stable`: Low volatility with minimal changes
- `consolidating`: Sideways movement within tight range

**cURL Example:**

```bash
curl -X GET \
  -H "X-API-Key: your-api-key-here" \
  https://crypto-ai.local:10443/api/market/tendency
```

```python
import requests

headers = {"X-API-Key": "your-api-key-here"}
response = requests.get(

    headers=headers
)

if response.status_code == 200:
    data = response.json()
    print(f"Market is {data['tendency']} with {data['confidence']*100}% confidence")
    print(f"Average change: {data['metrics']['avg_change_percent']}%")
```

## Code Examples

### Python SDK Example

```python
import requests
from typing import Dict, List, Optional

class CryptoAIClient:
    def __init__(self, api_key: str, base_url: str = "https://crypto-ai.local:10443/api"):
        self.api_key = api_key
        self.base_url = base_url
        self.headers = {
            "X-API-Key": api_key,
            "Content-Type": "application/json"
        }
    
    def get_top_predictions(self) -> Dict:
        """Get top 20 cryptocurrency predictions."""
        response = requests.get(
            f"{self.base_url}/predictions/top20",
            headers=self.headers
        )
        response.raise_for_status()
        return response.json()
    
    def get_market_tendency(self) -> Dict:
        """Get current market tendency."""
        response = requests.get(
            f"{self.base_url}/market/tendency",
            headers=self.headers
        )
        response.raise_for_status()
        return response.json()
    
    def ask_question(self, question: str, session_id: str) -> Dict:
        """Submit a question to the chat interface."""
        payload = {
            "question": question,
            "session_id": session_id
        }
        response = requests.post(
            f"{self.base_url}/chat/query",
            headers=self.headers,
            json=payload
        )
        response.raise_for_status()
        return response.json()

# Usage
client = CryptoAIClient(api_key="your-api-key-here")

# Get predictions
predictions = client.get_top_predictions()
print(f"Top performer: {predictions['predictions'][0]['symbol']}")

# Get market tendency
tendency = client.get_market_tendency()
print(f"Market is {tendency['tendency']}")

answer = client.ask_question(

    session_id="user-123"

print(f"Answer: {answer['answer']}")
```

## Best Practices

### 1. API Key Security

- **Never expose API keys** in client-side code or public repositories
- Store API keys in environment variables or secure vaults
- Rotate API keys regularly
- Use different API keys for development and production

### 2. Error Handling

Always implement proper error handling:

```python
try:
    response = requests.get(url, headers=headers)
    response.raise_for_status()
    data = response.json()
except requests.exceptions.HTTPError as e:
    if e.response.status_code == 429:
        # Handle rate limiting
        retry_after = e.response.headers.get('Retry-After', 60)
        time.sleep(int(retry_after))
    elif e.response.status_code == 401:
        # Handle authentication error
        print("Invalid API key")
    else:
        print(f"HTTP error: {e}")
except requests.exceptions.RequestException as e:
    print(f"Request failed: {e}")
```

### 3. Rate Limiting

Implement exponential backoff for rate limit errors:

```python
import time

def make_request_with_retry(url, headers, max_retries=3):
    for attempt in range(max_retries):
        try:
            response = requests.get(url, headers=headers)
            response.raise_for_status()
            return response.json()
        except requests.exceptions.HTTPError as e:
            if e.response.status_code == 429:
                wait_time = 2 ** attempt  # Exponential backoff
                time.sleep(wait_time)
            else:
                raise
    raise Exception("Max retries exceeded")
```

### 6. Monitoring

Track API usage and costs:

```python
class MonitoredClient(CryptoAIClient):
    def __init__(self, *args, **kwargs):
        super().__init__(*args, **kwargs)
        self.request_count = 0
        self.total_cost = 0.0
    
    def ask_question(self, question, session_id):
        result = super().ask_question(question, session_id)
        self.request_count += 1
        self.total_cost += result.get('cost_usd', 0)
        return result
    
    def get_stats(self):
        return {
            'requests': self.request_count,
            'total_cost': self.total_cost,
            'avg_cost': self.total_cost / max(self.request_count, 1)
        }
```
